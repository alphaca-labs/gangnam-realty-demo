## 0. 데모 핵심 메시지: "왜 웹버전이 필요한가"

> **클라이언트 핵심 관심사:** 카카오톡 챗봇은 **텍스트 데이터 입출력만** 가능.  
> 웹버전은 **데이터 외 입출력**(차트·지도·양식·계산기·서류 뷰어)으로  
> 사용률을 끌어올릴 수 있음을 시각적으로 증명해야 한다.

### 카카오톡 vs 웹 — 입출력 비교

| 기능 | 카카오톡 챗봇 | 웹버전 |
|------|:---:|:---:|
| 텍스트 질의/응답 | ✅ | ✅ |
| 인터랙티브 차트 (실거래가 추이) | ❌ | ✅ |
| 테이블 정렬/필터 | ❌ | ✅ |
| 중개수수료 계산기 (실시간) | ❌ (텍스트 안내만) | ✅ |
| 지도 위 매물 표시 | ❌ | ✅ |
| 서류 양식 뷰어/다운로드 | ❌ | ✅ |
| 전자계약 미리보기/서명 | ❌ | ✅ |
| SEO 검색 유입 | ❌ | ✅ |
| PC 넓은 화면 활용 | ❌ | ✅ |
| 복합 정보 비교 (사이드바이사이드) | ❌ | ✅ |

**데모에서 이 차이를 시각적으로 보여줘야 한다.**

> 💡 메인 랜딩에 "카카오톡에서는 텍스트로만 볼 수 있던 정보를,  
> 이제 차트·지도·계산기로 바로 확인하세요" 카피 배치

## 1. 데모 범위 & 우선순위

### 🔴 P0 — 반드시 동작 (인터랙티브)
| # | 화면 | 핵심 인터랙션 |
|---|------|---------------|
| 1 | **메인 랜딩** | AI 검색창 + 빠른 메뉴 + 히어로 |
| 2 | **AI 질의 → 리치 카드** | 질문 입력 → 실거래가 차트/테이블/가이드 카드 렌더링 |
| 3 | **중개수수료 계산기** | 매매/전세/월세 유형 선택 → 자동 계산 |
| 4 | **실거래가 조회** | 동/단지 선택 → 차트 + 테이블 뷰 |

### 🟡 P1 — 정적 목업 (UI만, 인터랙션 없음)
| # | 화면 | 비고 |
|---|------|------|
| 5 | **민원 가이드** | 거래신고/토지거래허가/중개업 개설/전세사기 예방 카드 |
| 6 | **지도 뷰** | 강남구 지도 + 더미 매물 핀 (정적 이미지 or 카카오맵 embed) |
| 7 | **서류/양식 가이드** | 중개계약 필수 입력 항목 + 견본 표시 |

### ⚪ P2 — 데모 제외 (본개발)
- 전자계약 (케이스 분기 미확정)
- 중개대상물 확인서 작성/관리
- 관리자(어드민) 페이지
- 회원가입/로그인
- 실제 AI 엔진 연동

### 2-3. 중개수수료 계산기

```
┌─────────────────────────────────────────┐
│ 🧮 중개수수료 계산기                     │
│                                         │
│ 거래유형                                 │
│ ┌──────┐ ┌──────┐ ┌──────┐             │
│ │ 매매 │ │ 전세 │ │ 월세 │             │
│ └──────┘ └──────┘ └──────┘             │
│                                         │
│ 거래금액                                 │
│ ┌───────────────────────────────┐       │
│ │ 500,000,000              원  │       │
│ └───────────────────────────────┘       │
│ [1억] [3억] [5억] [10억] (퀵버튼)       │
│                                         │
│ ── 월세인 경우 ──                        │
│ 보증금: [        ] 원                    │
│ 월세:   [        ] 원                    │
│                                         │
│ [계산하기]                               │
│                                         │
│ ┌─ 계산 결과 ────────────────────────┐  │
│ │  상한 요율:        0.4%            │  │
│ │  중개보수 상한:    2,000,000원     │  │
│ │  부가세(10%):      200,000원       │  │
│ │  ─────────────────────────         │  │
│ │  최대 총 비용:     2,200,000원     │  │
│ │                                    │  │
│ │  📌 서울특별시 조례 기준            │  │
│ │  매매 5억 미만: 0.4% 이내         │  │
│ │  (한도 200만원)                    │  │
│ └────────────────────────────────────┘  │
│                                         │
│ ┌─ 요율표 참고 ──────────────────────┐  │
│ │  거래금액 | 상한요율 | 한도액       │  │
│ │  5천만 미만 | 0.6% | 25만원       │  │
│ │  5천~2억   | 0.5% | 80만원       │  │
│ │  2억~9억   | 0.4% | -            │  │
│ │  9억~12억  | 0.5% | -            │  │
│ │  12억~15억 | 0.6% | -            │  │
│ │  15억 이상  | 0.7% | -            │  │
│ └────────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

**계산 로직 (서울특별시 조례 기준):**
```
매매:
  5천만 미만     → 0.6% (한도 25만)
  5천만~2억 미만  → 0.5% (한도 80만)
  2억~9억 미만   → 0.4%
  9억~12억 미만  → 0.5%
  12억~15억 미만 → 0.6%
  15억 이상      → 0.7%

전세:
  5천만 미만     → 0.5% (한도 20만)
  5천만~1억 미만  → 0.4% (한도 30만)
  1억~6억 미만   → 0.3%
  6억~12억 미만  → 0.4%
  12억~15억 미만 → 0.5%
  15억 이상      → 0.6%

월세:
  환산보증금 = 보증금 + (월세 × 100)
  → 환산보증금 기준으로 전세 요율 적용
```

**컴포넌트:**
- `TransactionTypeSelector` — 매매/전세/월세 탭
- `AmountInput` — 금액 입력 (포맷팅 + 퀵버튼)
- `CalculationResult` — 결과 카드 (애니메이션)
- `RateTable` — 요율표 참고 (아코디언)

### 2-5. 민원 가이드 (P1 — 정적)

4개 카테고리, 각각 스텝 바이 스텝:
1. **거래신고** — 부동산 거래 후 30일 이내 신고 절차
2. **토지거래허가** — 허가구역 내 거래 시 절차
3. **중개업 개설/폐업** — 공인중개사 사무소 개설 절차
4. **전세사기 예방** — 체크리스트 + 주의사항

### 2-6. 지도 뷰 (P1 — 정적)

- 카카오맵 API embed (강남구 중심)
- 더미 마커 10~15개 (주요 단지)
- 마커 클릭 → 간단한 팝업 (단지명, 평균가)

### 2-7. 서류/양식 가이드 (P1 — 정적)

- 중개계약서 필수 입력 항목 체크리스트
- 견본 이미지/PDF 뷰어
- 중개대상물 확인서 샘플

## 4. 디자인 가이드라인 (→ 영희 전달)

### 4-1. 브랜드 톤

- **관공서 + 테크** — 신뢰감 있으면서 현대적
- 강남구청 CI 색상 참고하되, 웹 최적화

### 4-2. 컬러 팔레트 (제안)

```

Surface:     #FFFFFF
Text:        #1A1A2E (다크)
Text-sub:    #64748B (서브텍스트)
Error:       #EF4444
```

### 4-3. 반응형 브레이크포인트

```
Mobile:   < 768px  (1열 레이아웃, 하단 탭 네비게이션)

Desktop:  > 1024px (사이드바 + 메인)
```

### 4-4. 핵심 디자인 원칙

1. **정보 계층** — 가장 중요한 정보가 가장 먼저 보이게

3. **관공서 신뢰감** — 강남구청 로고/인증 배지, 공식 느낌
4. **모바일 퍼스트** — 카카오톡에서 넘어오는 사용자 고려
